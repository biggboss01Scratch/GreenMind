# GreenMind

基于 STM32、ESP07 和电脑端大模型网关的智慧植物养护建议与辅助浇水系统。

## 当前阶段

项目目前已经完成 **传感器 → STM32 → ESP07 → Python 网关 → DeepSeek → TFT** 的真实最小闭环，正在进入稳定性验收和执行器安全设计阶段。

当前实现和验证情况：

- DHT11 和光敏数据已进入 STM32 状态与 TFT；
- ESP07 使用 AP+STA 模式连接电脑 2.4 GHz 热点，并提供 `:8080` TCP Server；
- Python 网关可完成 `HELLO / HELLO_ACK`、植物数据、AI 和图片双向协议；
- 网关具备 3 秒心跳和 12 秒失联检测，可恢复“仅重启 STM32、ESP07 保留旧 TCP”的连接；
- 真实 DeepSeek 最小闭环已完成过实物验证，TFT 能进入 `AI: DONE`；
- TFT 现有 4 个主页面和植物库/植物档案 2 个子页面，使用圆角 UI、触摸操作和局部刷新；
- 电脑端 SQLite 已保存 6 种植物、适宜区间、设备当前植物、图片索引和 AI 日志；
- 植物选择可从 TFT 发起并持久化到 `GM001.current_species_id`；
- 网关先执行确定性植物规则，再让 Mock 或 DeepSeek 解释，模型不能覆盖规则枚举；
- 已生成 6 种植物 × 3 种状态的 18 张 48×48 PNG/RGB565 资源；
- 绿萝三种状态图内置固件，薄荷、多肉、仙人掌、蝴蝶兰和番茄按需从数据库下载；
- 完整 GB2312 12×12 字库和 Unicode 映射已链接，六页面中文标签已实现；
- DeepSeek/Mock 可生成结合植物档案、当前环境和趣味知识的中文养护台词；
- 中文台词通过分包、序号、UTF-8、长度和 CRC32 校验下发到 STM32；
- AI 页面已实现植物头像、自动换行、触摸滚动和滚动条的 Galgame 风格“AI养护手记”；
- 中文字体资产与 134 条文案通过软件校验，英文回退开关已通过编译验证；
- 最新固件完整重建为 0 错误、7 个原 TFT 驱动未使用变量警告；
- Python 网关 38 项自动化测试全部通过。

本次中文版本和 AI 长文本对话框尚待烧录后的实机验收。电脑端分包闭环、
CRC 重组、数据库记录、真实 DeepSeek 中文输出和固件构建均已通过软件验证。
原有安全枚举仍然保留，长文本传输失败时不会影响 `AI_RESULT`。

热点密码、模型 API Key 等秘密信息不写入本文档。后续统一通过本地配置文件或环境变量管理。

## MVP 总体链路

```text
DHT11 / 光敏 / 按键
          ↓
        STM32
          ↓ USART3
        ESP07
          ↓ WiFi / TCP
电脑热点 + Python 网关
          ↓ HTTPS
       大模型 API
          ↓
安全状态码 + 个性化中文台词返回 STM32
          ↓
TFT 中文界面
          ↓
用户阅读建议
```

## 实施原则

1. `wifi_tft` 等已验证工程作为只读基线，不直接堆叠综合业务。
2. 在 `GreenMind` 中建立新的固件和电脑网关代码。
3. 传感器、网络、协议、UI、水泵分别封装，主循环只负责调度和状态流转。
4. 先使用固定响应打通链路，再接入真实模型。
5. 大模型只给建议，不能直接控制水泵。
6. WiFi、电脑或模型失败时，本地采集、显示和手动停止水泵必须继续工作。
