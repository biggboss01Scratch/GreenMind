# GreenMind 植物数据库与像素资源记录 v1

## 1. 本阶段范围

本阶段只建立电脑端数据库和图片资源基础，不修改 STM32 固件、现有
`AI_REQ` 格式或 DeepSeek 调用流程。

已完成：

- SQLite 模式和可重复初始化脚本；
- 6 种植物基本信息及环境偏好；
- 默认设备 `GM001`，默认植物为绿萝 `pothos`；
- 每种植物 `NORMAL / ATTENTION / DANGER` 三种状态图；
- PNG 预览和 STM32 可直接使用的 RGB565 原始数据；
- 图片尺寸、字节数、CRC32、SHA-256 校验；
- 植物查询、当前植物更新和 AI 分析记录接口；
- 数据库与资源自动化测试。

尚未完成：

- STM32 植物列表、详情和选择页面；
- 图片分块、流控、进度和 CRC 通信；

本记录创建后的后续进展：

- 已增加兼容消息 `PLANT_AI_REQ`；
- 当前固件默认上传 `GM001/pothos`；
- 数据库档案已参与 Mock 和 DeepSeek 分析；
- 规则层先确定状态码，大模型只能生成解释；
- TFT AI 页面显示 `GM001/POTHOS`。

0～100 相对值，不是 lux。实物使用前仍需校准。

## 3. 数据库设计

数据库源文件：

```text
gateway/database/schema.sql
gateway/database/seed_plants.json
```

运行数据库：

```text
gateway/data/greenmind.sqlite3
```

运行数据库由脚本生成并被 `.gitignore` 排除。Git 保存可审查的模式、种子数据
和资源清单，避免直接维护不可读的 SQLite 二进制文件。

表：

```text
schema_meta
plant_species
plant_requirements
plant_images
devices
ai_analysis_logs
```

`ai_analysis_logs` 会保存请求时使用的 `species_id`，后续切换植物不会修改旧分析
记录的植物身份。

## 4. 像素资源规范

每张图片：

```text
显示尺寸：48 × 48
逻辑像素：16 × 16
整数缩放：3
预览格式：PNG RGB
设备格式：RGB565_BE
原始大小：4608 bytes
字节顺序：高字节在前
```

三种状态：

| 状态 | 含义 |
|---|---|
| `NORMAL` | 当前环境符合所选植物档案 |
| `ATTENTION` | 一个或多个环境指标需要关注 |
| `DANGER` | 严重偏离或传感器故障 |

状态图是系统状态可视化，不代表对真实植物病害作出诊断。

资源目录：

```text
gateway/assets/plants/
```

`catalog_preview.png` 按六列三行排列：

```text
列：pothos / mint / succulent / cactus / orchid / tomato
行：NORMAL / ATTENTION / DANGER
```

数据库不保存图片 BLOB，只保存相对路径、版本、尺寸和校验值。这样便于替换
资源、Git 审查和网关分块读取。

## 5. 重建与验证

在 `gateway` 目录执行：

```powershell
python .\tools\generate_plant_assets.py
python .\tools\init_database.py --reset
python -m unittest discover -s .\tests -v
```

初始化会拒绝以下资源：

- 不属于 6 种已知植物；
- 缺少任意状态；
- 尺寸不是 48×48；
- RGB565 原始大小不是 4608 字节；
- CRC32 或 SHA-256 不匹配；
- 路径越出 `gateway/assets`。

## 6. 当前接入结果

- `pothos` 的三种状态图内置固件，可离线显示；
- `mint / succulent / cactus / orchid / tomato` 均使用数据库图片记录；
- 用户选择在线植物后，STM32只请求当前状态的一张图；
- AI 状态变化时按 `NORMAL / ATTENTION / DANGER` 请求对应图片；
- STM32只缓存一张 48×48 RGB565 图片，不会同时加载全部 18 张；
- 资源、数据库、固件策略和 Python 38 项测试均已通过软件验证；
- 五种在线植物的逐一下载和状态切换仍需实机验收。
