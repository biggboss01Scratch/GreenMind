# GreenMind 圆角 UI 优化记录 v1

日期：2026-07-17

## 优化目标

在 STM32F103、320×480 TFT、无整屏帧缓冲的条件下，提高四页界面的完成度和课程展示效果，同时保留现有局部刷新、触摸、WiFi 和 AI 状态机。

## 已完成的视觉优化

- 页面顶部改为悬浮圆角标题栏。
- 主内容区改为带轻量阴影的圆角卡片。
- HOME 中的像素绿植放入浅绿色圆角展示区。
- 环境总结改为胶囊状态标签。
- `DETAIL / ASK AI / ANALYZE / RETRY / CHECK / CALIBRATE` 改为圆角按钮。
- DETAIL 的三条传感器进度条改为圆角进度条。
- AI 的四段调用进度改为圆角进度块。
- Gateway 状态改为彩色胶囊标签。
- 底部导航改为悬浮圆角 Dock，当前页面使用深绿色圆角选中块。
- 统一使用浅灰绿色背景、白色卡片、深绿色主色、黄色警告和红色错误。

## 实现方式

新增的通用绘制组件包括：

```c
UiService_FillRoundRect()
UiService_Card()
UiService_Button()
UiService_Pill()
UiService_ProgressBar()
```

圆角使用整数扫描线计算，不使用浮点数，不保存完整屏幕图像，也不需要 GPU。

阴影不是透明模糊，而是在卡片和按钮后方绘制一个浅色偏移圆角矩形。这种方法开销小，在 RGB565 屏幕上能提供清晰的层次感。

## 性能与刷新策略

- 圆角页面结构只在进入页面时完整绘制。
- 温度、湿度、光照变化时，只更新对应文字和进度条。
- WiFi 状态只更新右上角状态点或系统字段。
- AI 状态变化只更新像素绿植、状态胶囊、进度块和建议区域。
- 触摸坐标区域没有改变，原四页导航和按钮逻辑继续有效。
- 不使用透明混合、实时模糊和整屏动画。

## 板端验证重点

1. 检查标题栏、主卡片、按钮和底部导航是否呈现圆角。
2. 检查圆角边缘是否仍保持正确方向，没有重新出现左右镜像。
3. 依次切换 HOME、DETAIL、AI、SYSTEM，观察页面重绘。
4. 停留在 HOME，等待传感器或 WiFi 状态变化，确认没有整屏闪烁。
5. 调用 AI，确认像素绿植思考状态和 AI 圆角进度块正常更新。
6. 检查按钮触摸区域与视觉按钮一致。
7. 检查 SYSTEM 页面仍显示 `PUMP: NOT CONNECTED`。

## 保留的限制

- 阴影为色块模拟，不是手机式模糊阴影。
- 字体仍是点阵字体，没有手机级抗锯齿。
- 页面切换会完整绘制一次，不做高帧率滑动动画。
- 没有半透明、毛玻璃或实时渐变。
