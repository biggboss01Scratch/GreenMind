# GreenMind 植物选择闭环记录 v1

日期：2026-07-17

## 本阶段结果

这一版不再只是把默认植物名写死为 `POTHOS`，而是实现了由 SQLite 驱动的植物选择闭环：

1. STM32 与 Python 网关握手成功后，自动同步植物列表和数据库中保存的当前植物。
2. 点击主页中央的植物卡片，进入 `PLANT LIBRARY`。
3. 植物库显示 6 种植物以及 `BUILTIN / ONLINE / CURRENT` 状态。
4. 点击植物卡片，进入 `PLANT PROFILE`，查看温度、空气湿度和相对光照范围。
5. 点击 `USE PLANT`，网关先更新 SQLite 的 `devices.current_species_id`。
6. STM32 只有收到 `PLANT_SELECTED` 确认后，才更新主页植物名和后续 AI 请求中的 `species_id`。
7. 选择成功后会清空上一种植物的 AI 结果，避免把旧建议误显示为新植物的建议。

本文记录的是植物选择闭环首次完成时的状态；当时只使用代码绘制的小图标和文字卡片。后续已经加入绿萝内置资源和 RGB565 按需传输，参见 `植物图片加载闭环记录_v1.md`。

## 页面流程

```text
HOME
  └─ 点击中央植物卡
      └─ PLANT LIBRARY
          ├─ BACK
          ├─ REFRESH
          └─ 点击任意植物
              └─ PLANT PROFILE
                  ├─ BACK
                  └─ USE PLANT
                      └─ 收到数据库确认后返回 HOME
```

底部 `HOME / DETAIL / AI / SYSTEM` 导航在植物子页面仍然可用。

## 新增协议

STM32 请求植物列表：

```text
V1|PLANT_LIST_REQ|GM001
```

网关返回：

```text
V1|PLANT_LIST_BEGIN|6|pothos|POTHOS
V1|PLANT_ITEM|0|mint|MINT|BUILTIN
V1|PLANT_ITEM|1|pothos|POTHOS|BUILTIN
...
V1|PLANT_LIST_END|6
```

`PLANT_LIST_BEGIN` 同时携带 SQLite 中保存的当前植物，因此断电重启并重新连接网关后，固件会恢复数据库中的选择。

STM32 请求植物档案：

```text
V1|PLANT_DETAIL_REQ|cactus
```

网关返回：

```text
V1|PLANT_DETAIL|cactus|CACTUS|18|35|20|55|65|100|ONLINE
```

六个数字依次是温度最小/最大值、空气湿度最小/最大值、相对光照最小/最大值。

STM32 提交选择：

```text
V1|PLANT_SELECT|GM001|cactus
```

成功：

```text
V1|PLANT_SELECTED|cactus|CACTUS
```

失败：

```text
V1|PLANT_ERROR|DEVICE_NOT_FOUND
V1|PLANT_ERROR|PLANT_NOT_FOUND
V1|PLANT_ERROR|DATABASE_ERROR
```

植物列表按 20 ms 间隔逐帧发送，降低 ESP07/STM32 仅 128 字节 TCP 接收环形缓冲区发生突发溢出的概率。固件协议事件队列由 8 槽增加到 16 槽。

## 实物验证步骤

1. 烧录 `firmware/Obj/GreenMind.hex`。
2. 等 TFT 显示 WiFi、TCP 和 Gateway 均已连接。
3. 在电脑进入 `GreenMind\gateway`，启动 Mock 网关：

```cmd
cd /d "D:\2026SecondHalf\Puzhong\普中开发板\GreenMind\gateway"
python .\run_gateway.py 屏幕显示的STA-IP --provider mock
```

4. 点击主页中央的像素植物卡片。
5. 确认植物库出现 `MINT / POTHOS / CACTUS / ORCHID / SUCCULENT / TOMATO`。
6. 点击 `CACTUS`，确认档案显示 `T 18-35 / H 20-55 / L 65-100`。
7. 点击 `USE PLANT`，确认自动返回主页并显示 `CACTUS`。
8. 再次进入植物库，确认 `CACTUS` 标记为 `CURRENT`。
9. 点击 `ASK AI`，网关日志应包含 `plant=cactus`。
10. 重启开发板并重新连接网关，再进入植物库，确认仍同步为 `CACTUS`。
