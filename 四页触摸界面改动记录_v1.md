# 智慧农业四页触摸界面改动记录 v1

日期：2026-07-17

## 1. 本次目标

在不破坏已经验证成功的 ESP07、TCP、Python 网关和 DeepSeek 调用链路的前提下，将原来的两页按键界面升级为四页触摸界面。

本版本仍不控制水泵。AI 只提供建议，固件不会发送继电器或水泵启动指令。

## 2. 四个页面

### HOME

- 显示温度、湿度和光照。
- 显示 WiFi、TCP、AI 网关摘要。
- 显示最近一次 AI 状态和建议代码。
- `ASK AI` 可直接发起一次 AI 请求。
- AI 正在运行时按钮显示 `AI WORKING`，避免重复请求。

### AI

- 显示 `SENDING / PREPARING / THINKING / VALIDATING / DONE / ERROR`。
- 用四段进度条显示调用阶段。
- 显示请求编号、状态、主要问题、浇水建议、行动建议和错误。
- `ANALYZE` 发起新请求。
- `RETRY` 在调用失败或结束后重新发起请求。
- AI 忙碌期间两个按钮禁用。

### STATUS

- 显示 WiFi 状态和目标热点名称。
- 显示 ESP07 的 STA IP。
- 显示 TCP 和 Gateway 状态。
- 显示最近一次错误。
- WiFi 离线时，`RETRY WIFI` 请求 STA 重新扫描和连接。
- WiFi 已连接时，按钮显示 `CHECK AGAIN`。若 Gateway 离线，固件提示 `WAIT FOR GATEWAY`，此时应启动或检查电脑 Python 网关。

### CTRL

- 明确显示 `PUMP: NOT CONNECTED`。
- 显示 `OUTPUT COMMANDS DISABLED` 和 `AI PROVIDES ADVICE ONLY`。
- 不产生任何水泵控制输出。
- `CALIBRATE TOUCH` 可手动重新进行四点触摸校准。

## 3. 导航与备用按键

屏幕底部固定显示四个触摸标签：

```text
HOME | AI | STATUS | CTRL
```

触摸标签在松手时触发，长按不会连续切页。

原实体按键继续保留：

- `KEY_UP`：依次循环四个页面。
- `KEY1`：发起 AI 请求。

## 4. 局部刷新策略

- 页面切换时完整绘制一次新页面。
- 同一页面内，传感器、网络和 AI 状态改变时只刷新对应字段。
- AI 进度、建议和错误区域按状态变化更新。
- 不因周期性传感器采样而整屏清屏。

因此，正常状态变化不应再出现明显整屏闪烁；主动切换页面时出现一次页面重绘属于正常现象。

## 5. 触摸校准

触摸屏使用与 `34-触摸屏实验` 相同的镜像修正和 EEPROM 校准格式，校准标记为 `0x0B`。

- 如果刚才已经在同一块板上完成 34 实验校准，本工程通常直接读取参数，不再要求校准。
- 如果 EEPROM 中没有有效参数，首次启动会进入四点校准。
- 可在 `CTRL` 页面点击 `CALIBRATE TOUCH` 手动重新校准。

## 6. 引脚资源核对

| 模块 | 引脚 |
| --- | --- |
| ESP07 USART3 | PB10、PB11 |
| ESP07 使能 | PA4、PA15 |
| 电阻触摸 | PB1、PB2、PF9、PF10、PF11 |
| 校准 EEPROM I²C | PB6、PB7 |
| 光敏传感器 | PF8 / ADC3 |
| DHT11 | PG11 |
| TFT 背光 | PB0 |

本次新增触摸和 EEPROM 引脚与当前 ESP07、光敏、DHT11 引脚没有直接冲突。

## 7. 代码结构

新增或接入的主要模块：

```text
APP/agriculture/touch_service.c  触摸松手事件和按钮命中
APP/touch/touch.c                电阻触摸读取与校准
APP/24Cxx/24cxx.c                校准参数持久保存
APP/iic/iic.c                    EEPROM 软件 I²C
APP/agriculture/ui_service.c     四页界面与局部刷新
```

`main.c` 只负责调度触摸事件，并复用原有的 AI 请求、WiFi 状态机和协议处理函数。DeepSeek API Key 仍只保存在电脑网关侧。

## 9. 板端验证步骤

1. 烧录 `GreenMind/firmware/Obj/GreenMind.axf`。
2. 检查画面方向正常，且没有左右镜像。
3. 依次触摸底部四个标签，确认四页均可进入。
4. 在 HOME 点击 `ASK AI`，确认自动切到 AI 页面。
5. 启动 Python 网关，确认 AI 页面依次显示 `THINKING`、`VALIDATING` 和 `DONE`。
6. 关闭 Python 网关，确认 STATUS 显示 `GATEWAY: OFFLINE`，界面和传感器继续运行。
7. 关闭热点后点击 `RETRY WIFI`，确认重新进入扫描流程。
8. 在 CTRL 页面确认水泵始终为 `NOT CONNECTED`。
9. 只有触点偏移时才点击 `CALIBRATE TOUCH`，完成四点校准。

## 10. 当前边界

- 仍使用英文和结构化建议代码。
- 没有软键盘和屏幕 WiFi 密码输入。
- 没有历史曲线和数据存储。
- 没有手机 App。
- 没有水泵驱动和自动控制。
- `STATUS` 页不能代替电脑网关；WiFi 已连接但 Gateway 离线时，仍需在电脑启动 `run_gateway.py`。
