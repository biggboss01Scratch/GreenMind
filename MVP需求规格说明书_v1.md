# GreenMind MVP 需求规格说明书 v1

## 1. 文档定位

- 版本：v1.0
- 日期：2026-07-16
- 状态：实现基线
- 依据：现有开发板源码、独立模块实验和 WiFi 实物联调结果

本文件是第一版 MVP 的实现与验收依据。`初步需求文档.md` 中与本文件冲突的内容，以本文件为准。

## 2. MVP 目标

实现一个可以在网络异常时独立运行、在网络正常时请求大模型建议的植物环境监测与辅助浇水系统。

最小闭环：

```text
采集温度、空气湿度和光照
→ TFT 显示本地状态
→ 用户按键请求 AI 分析
→ STM32 通过 WiFi/TCP 上传数据
→ 电脑网关调用大模型
→ STM32 接收经过校验的建议码
→ TFT 显示简短建议
→ 用户长按按键启动水泵
→ STM32 到时自动关闭水泵
```

## 3. MVP 范围

### 3.1 必须包含

- DHT11 整数温度、空气湿度采集；
- 板载光敏传感器相对亮度采集；
- TFT 实时数据页、AI 建议页和故障状态；
- ESP07 连接电脑热点；
- STM32 与电脑网关 TCP 全双工通信；
- 用户主动发起 AI 分析；
- 电脑网关调用模型并校验结果；
- 按键手动启动水泵、本地定时关闭和紧急停止；
- 至少一种本地异常提示；
- 网络和模型不可用时保持本地功能。

### 3.2 第一版不包含

- 自动浇水决策；
- 土壤湿度检测；
- 任意动态中文字体显示；
- 手机 App；
- 云服务器和远程控制；
- 图像识别、数据库、日报和多植物管理；
- 模型直接控制水泵、电机或任意 GPIO。

彩灯和蜂鸣器属于增强项，在核心闭环稳定后接入。

## 4. 实际系统结构

```text
STM32 ─USART3→ ESP07 ─WiFi→ 电脑热点
  ↑                    │
  └──── TCP :8080 ─────┘

电脑 Python 网关：
TCP Client ←→ STM32/ESP07 TCP Server
HTTPS      ←→ 大模型 API
```

### 4.1 网络角色

| 对象 | 角色 |
|---|---|
| Windows 电脑 | 2.4 GHz 热点 AP，同时运行 Python 网关 |
| ESP07 | STA 连接电脑热点；当前保留 AP+STA 模式作为调试回退 |
| ESP07 TCP | Server，监听 STA IP 的 8080 端口 |
| Python 网关 | TCP Client，主动连接 TFT 显示的 STA IP |
| 大模型连接 | Python 网关通过 HTTPS 发起 |

STM32 不直接构造 HTTP 请求，也不解析模型 API 的大段 JSON。

## 5. 硬件和引脚基线

| 模块 | 实际接口 | MVP 用途 | 注意事项 |
|---|---|---|---|
| DHT11 | PG11 | 整数温度、空气湿度 | 采集周期不得短于 1 秒，MVP 使用 2 秒 |
| 光敏 LS1 | PF8 / ADC3_CH6 | 0～100 相对亮度 | 不是 0～4095；阈值需实物校准 |
| ESP07 | PB10/PB11 / USART3 | WiFi/TCP | PA4、PA15 保持模块使能；P10 跳帽正确连接 |
| TFT | FSMC，PB0 背光 | 数据、AI、故障页面 | 动态中文能力有限 |
| 按键 | PA0、PE2、PE3、PE4 | 页面、AI、水泵、停止 | 具体映射放入统一配置 |
| 水泵驱动 | PB5 / TIM3_CH2 PWM | 手动浇水 | PB5 与 LED1 共用；不得用 GPIO 直接驱动水泵 |
| 5×5 WS2812 | PE5 | 状态颜色（增强项） | 刷新时会短暂影响中断，避免连续高频动画 |
| 蜂鸣器 | PB8 | 操作/故障提示（增强项） | 使用非阻塞节奏状态机 |

未发现以上 MVP 必选模块之间存在不可解决的直接引脚冲突。

## 6. 功能需求

### FR-01 系统启动

系统上电后依次完成基础时钟、TFT、传感器、按键、水泵安全输出和 WiFi 初始化。水泵的默认状态必须为关闭。

启动失败的模块应记录故障状态；除无法安全初始化水泵外，单个模块失败不得让主循环永久阻塞。

### FR-02 温湿度采集

- 每 2 秒读取一次 DHT11；
- 保存整数温度和整数相对湿度；
- 连续读取失败时设置 `DHT_ERROR`；
- 无效数据不能进入 AI 请求。

### FR-03 光照采集

- 每 500 毫秒至 1 秒读取一次；
- 使用现有驱动的 0～100 相对亮度；
- 初始状态阈值暂定：`0～29 DARK`、`30～69 NORMAL`、`70～100 STRONG`；
- 阈值必须集中配置，实物校准后可直接修改，不散落在 UI 和网络代码中。

### FR-04 本地状态判断

MVP 默认规则：

| 数据 | 正常 | 注意 | 严重/故障 |
|---|---|---|---|
| 温度 | 10～30℃ | 低于 10℃或 31～35℃ | 高于 35℃或读取失败 |
| 空气湿度 | 35～75% | 低于 35%或高于 75% | 读取失败 |
| 光照 | 30～69 | 0～29 或 70～100 | ADC/采集故障 |

规则只做本地提示，不直接决定浇水。

### FR-05 TFT 页面

至少提供：

1. 实时数据页：温度、空气湿度、光照、WiFi、系统状态；
2. AI 建议页：请求状态、环境级别、主要问题、浇水建议码和固定短句；
3. 故障覆盖层或故障页：传感器、网络、模型、水泵相关错误。

TFT 第一版使用 ASCII 和英文预置短句。完整英文建议保留在电脑端日志或界面。若后续必须显示中文，使用“建议码 → 固定中文字模”映射，不直接显示模型返回的任意中文。

### FR-06 WiFi 连接

- ESP07 连接电脑 2.4 GHz 热点；
- TFT 显示 `SEARCHING`、`CONNECTING`、`CONNECTED`、`DISCONNECTED` 和 STA IP；
- 断开后自动重试；
- WiFi 失败不得影响本地传感器、页面切换和水泵停止。

热点 SSID、密码、端口和超时必须集中在配置模块。密码不得写入公开文档。

### FR-07 TCP 全双工

- ESP07 监听 TCP 8080；
- Python 网关主动连接 STA IP；
- STM32 可以主动上传数据，也可以接收电脑回复；
- 收发必须使用主循环状态机，不在 USART3 中断中等待；
- 正式协议必须处理 TCP 拆包、粘包、超长帧和断线；
- 单条 MVP 应用帧不超过 120 个 ASCII 字节，以 `CRLF` 结束。

详细格式见 `TCP通信协议_v1.md`。

### FR-08 AI 请求

用户按下 AI 分析键后：

1. 检查传感器数据有效性；
2. 检查 TCP 网关连接；
3. 生成唯一请求编号；
4. 发送当前温度、空气湿度、亮度值和光照等级；
5. TFT 显示 `AI ANALYZING`；
6. 等待最多 45 秒；
7. 收到匹配编号的结果后更新 AI 页面；
8. 超时或格式错误时显示失败，但继续本地运行。

同一时间只允许存在一个未完成的 AI 请求。MVP 不自动周期调用模型。

### FR-09 电脑网关

电脑网关必须：

- 自动或手动连接配置的 ESP STA IP:8080；
- 解析版本化 TCP 帧；
- 先支持固定假响应，再切换真实模型；
- 将 API Key 从环境变量读取；
- 校验模型输出枚举和文本长度；
- 不向 STM32 下发任何水泵启动命令；
- 保存请求、原始模型结果、校验结果和错误日志；
- 断线后自动重连，不能因一条坏消息退出。

### FR-10 模型输出

电脑端内部模型结果采用结构化数据：

```json
{
  "status": "warning",
  "main_issue": "high_temperature_and_strong_light",
  "watering_advice": "check_soil",
  "advice_code": "move_to_shade",
  "suggestion_en": "Temperature and light are high. Avoid direct sunlight and check the soil."
}
```

Python 网关验证后只把短枚举和建议码发送给 STM32。`suggestion_en` 留在电脑端。

### FR-11 按键建议映射

第一版建议映射如下，最终在硬件集成前确认：

| 按键 | 操作 |
|---|---|
| KEY_UP / PA0 | 切换实时数据页与 AI 页 |
| KEY1 / PE3 | 请求 AI 分析 |
| KEY0 / PE4 长按 | 启动一次浇水 |
| KEY2 / PE2 | 立即停止水泵/取消当前操作 |

按键扫描周期不大于 20 毫秒，长按阈值暂定 1.5 秒，不使用长时间 `delay_ms` 判断长按。

### FR-12 水泵安全

- 上电默认关闭；
- 仅 KEY0 长按可以启动；
- 单次运行时间默认 3 秒；
- 到时由 STM32 本地定时逻辑强制关闭；
- 运行期间重复启动无效；
- KEY2 可立即停止；
- WiFi、电脑、模型或传感器故障时仍能停止；
- 模型协议中不存在 `PUMP_ON`、运行秒数或远程启动字段。

## 7. 软件结构要求

固件至少拆分为：

```text
app_config       可调参数和引脚映射
app_state        传感器、网络、AI、水泵统一状态
sensor_service   DHT11 和光敏调度
wifi_transport   ESP AT 和 TCP 收发
protocol         换行分帧、编码、解码和校验
ui_service       页面和局部刷新
key_service      消抖、短按、长按事件
pump_service     PWM、定时关闭和紧急停止
```

电脑网关至少拆分为：

```text
config           IP、端口、模型和超时
tcp_client       连接、收发和重连
protocol         与固件一致的协议
model_provider   假模型/真实模型统一接口
validator        模型结构和枚举校验
main             业务编排
```

禁止把模型调用、TCP 字节解析、屏幕绘制和水泵控制全部写入同一个主文件。

## 8. 非功能需求

| 项目 | 要求 |
|---|---|
| 主循环 | 正常运行阶段不得出现无限等待；任务使用状态机和时间戳 |
| 按键响应 | 不超过 200 毫秒；停止水泵优先处理 |
| 传感器刷新 | DHT11 2 秒；光敏不慢于 1 秒 |
| AI 超时 | 45 秒，超时后恢复可再次请求状态 |
| TCP | 断线可恢复；坏帧不导致死机 |
| RAM | 使用固定大小缓冲区，不使用动态内存分配 |
| API Key | 仅在电脑环境变量或未提交的本地配置中 |
| 可追踪性 | 每次硬件验证结果写入 `开发与验证记录.md` |

WS2812 第一版只在状态变化时刷新整屏颜色，避免持续动画影响 USART3 接收。

## 9. 状态模型

系统顶层状态建议分开保存，而不是使用一个混合枚举：

```text
SensorState: INIT / OK / ERROR
WiFiState:   SEARCHING / CONNECTING / CONNECTED / ERROR
TcpState:    WAITING / CONNECTED / SENDING / ERROR
AiState:     IDLE / REQUESTING / SUCCESS / TIMEOUT / ERROR
PumpState:   OFF / RUNNING / STOPPING / ERROR
```

这样后续加入土壤湿度、风扇或补光灯时，不需要重写整个状态机。

## 10. 验收标准

### AC-01 本地采集

TFT 连续显示合理的整数温度、空气湿度和 0～100 光照值；遮挡光敏后数值和等级发生变化。

### AC-02 网络

ESP07 能连接电脑热点，TFT 显示 STA IP；电脑可以连接该 IP 的 8080 端口。

### AC-03 正式协议

连接后完成 `HELLO/HELLO_ACK`；AI 请求带请求编号，返回编号匹配；拆分发送或连续发送两帧时解析仍正确。

### AC-04 固定 AI 闭环

在不调用真实模型时，Python 网关返回固定 `AI_RESULT`，STM32 能显示对应状态和建议码。

### AC-05 真实模型闭环

用户按键后，电脑日志显示模型请求，45 秒内 STM32 收到经过校验的结果并更新页面。正常网络下目标响应时间为 15 秒以内。

### AC-06 网络异常

关闭网关或热点后系统不死机；传感器和页面继续更新；恢复后可以重新连接并再次请求。

### AC-07 水泵

长按启动，3 秒后自动停止；运行期间重复启动无效；KEY2 可以立即停止。

### AC-08 AI 权限

构造包含水泵控制字段的模型结果时，Python 网关拒绝该字段，STM32 不启动水泵。

### AC-09 故障数据

DHT11 读取失败时不发送 AI 请求，TFT 显示传感器错误，本地停止水泵仍有效。

## 11. 实施顺序

1. 从 `wifi_tft` 复制建立独立 `firmware`，保持回退工程不变；
2. 将测试协议升级为正式换行分帧和请求编号；
3. 建立 Python 网关，先返回固定 AI 结果；
4. 合入 DHT11、光敏和实时数据页；
5. 打通“按键 → 数据上传 → 固定建议 → TFT”；
6. 接入真实模型并完成输出校验；
7. 合入按键长按和水泵安全状态机；
8. 最后加入彩灯和蜂鸣器提示。

每完成一步都必须重新验证已有功能，不同时引入多个未验证模块。
