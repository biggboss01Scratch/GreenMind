# 智慧农业四页展示界面改动记录 v2

日期：2026-07-17

## 修订原因

第一版四页界面优先实现了功能和故障诊断，但没有落实课程展示方案中最重要的“拟人化像素绿植主页”。v2 按 `chat建议v2.md` 重新调整页面结构和视觉重点。

## 四页最终定义

### 1. HOME：GreenMind 拟人化主页

- 左上角显示 `GreenMind`。
- 右上角显示 `WiFi` 和红/绿像素状态点。
- 中央显示由 LCD 色块实时绘制的拟人化像素绿植。
- 显示温度、湿度、光照和 AI Ready/Offline。
- 显示环境总结和最近一次主要建议。
- 提供 `DETAIL` 和 `ASK AI` 两个触摸按钮。

像素绿植不是固定图片，会根据系统状态改变：

| 状态 | 绿植表现 |
| --- | --- |
| 环境正常 | 绿色叶片、微笑表情、`ENVIRONMENT GOOD` |
| 温湿度或光照需要关注 | 黄绿色叶片、严肃表情、`PLANT NEEDS ATTENTION` |
| 传感器无效 | 灰色叶片、叉眼表情、`CHECK THE SENSORS` |
| AI 正在调用 | 思考表情和三个像素气泡、`AI IS THINKING...` |

### 2. DETAIL：环境详情

- 分别显示温度、湿度和光照。
- 每个传感器使用横向进度条。
- 显示 `LOW / NORMAL RANGE / HIGH / SENSOR ERROR`。
- 显示综合环境状态。
- 当前没有历史数据库，因此不伪造趋势箭头或历史曲线。

### 3. AI：植物 AI 助手

- 显示请求编号和调用状态。
- 显示四段调用进度。
- 显示主要问题、浇水建议、行动建议和错误。
- 提供 `ANALYZE` 和 `RETRY`。
- AI 忙碌时按钮禁用，防止重复请求。

### 4. SYSTEM：系统与安全

- 显示 WiFi、SSID、STA IP、TCP 和 Gateway。
- 显示 `PUMP: NOT CONNECTED`。
- 明确显示水泵输出命令已禁用。
- 提供 `RETRY WIFI` 和 `CALIBRATE`。
- WiFi 已连接但 Gateway 离线时，仍需检查电脑上的 Python 网关。

## 页面导航

底部固定四个触摸标签：

```text
HOME | DETAIL | AI | SYSTEM
```

此外：

- HOME 的 `DETAIL` 直接进入环境详情。
- HOME 的 `ASK AI` 发起 AI 请求并切换到 AI 页面。
- `KEY_UP` 仍可循环切换四页。
- `KEY1` 仍可发起 AI 请求。

## 刷新策略

- 页面切换时完整绘制新页面。
- 同一页面内仅刷新发生变化的数字、文字、状态点、进度条或绿植区域。
- WiFi 状态变化不会导致整屏清屏。
- AI 阶段变化只更新绿植状态或 AI 页面相应区域。

## 触摸与引脚

继续复用 `34-触摸屏实验` 已验证的电阻触摸驱动、X 方向镜像修正和 EEPROM 校准参数。

| 模块 | 引脚 |
| --- | --- |
| ESP07 | PB10、PB11、PA4、PA15 |
| 电阻触摸 | PB1、PB2、PF9、PF10、PF11 |
| 校准 EEPROM | PB6、PB7 |
| 光敏传感器 | PF8 |
| DHT11 | PG11 |

没有新增直接引脚冲突。

## 板端重点验证

1. 烧录 `firmware/Obj/Template.axf`。
2. 检查默认首页是否显示 `GreenMind`、WiFi 状态点和像素绿植。
3. 正常传感器数据下，绿植应显示微笑。
4. 点击 `DETAIL`，检查三条进度条和实时数值。
5. 点击 `ASK AI`，检查自动进入 AI 页面并显示调用阶段。
6. AI 调用过程中返回 HOME，检查绿植是否显示思考气泡。
7. 依次检查底部 `HOME / DETAIL / AI / SYSTEM`。
8. SYSTEM 页必须始终显示 `PUMP: NOT CONNECTED`，不能产生水泵控制。
